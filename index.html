This is a project to develop technology to quantify both magnitude and shape of lactation curves in a consistent and repeatable way. It is based on the simple observation that lactation curves vary, in consistent patterns, no doubt influenced by genetics, environment, and health. Tools for characterizing that variation are likely to be useful in understanding many problems in dairy herd management and dairy research.

Variability in lactation curves may be characterized by three observable characteristics of the curve plotting milk production against time for a lactation. These are

  • scale, the overall level of milk production
  • ramp, the rate of the rise in early lactation
  • decay the rate of exponential decline, most apparent in late lactation

Using MilkBot technology, we can quantify these observable characteristics as parameter values obtained by fitting production data from individual lactations to the MilkBot model. These parameter values can be treated as individual metrics as described in the paper Quantifying inter-group variability in lactation curve shape and magnitude with the MilkBotÂ® lactation model. Applications tend to fall into two classes, evaluating genetic merit of cattle, or evaluating the effects of management on production.

There are two essential components to MilkBot technology.

  1. The MilkBot Model

    This is a simple nonlinear algebraic model predicting daily milk production as a function of DIM and the four MilkBot parameters, scale, ramp, offset, and decay. From fitted parameter values it is easy to calculated daily production at any day in the curve, or M305, persistency, peak milk, peak day, and several other metrics.

  2. The MilkBot Fitting Engine

    MilkBot uses a nonlinear model, so the common methods for fitting linear models do not work. We have developed an iterative Bayesian fitting engines that can fit individual lactation using any number of data points yielding estimates of parameter values for that lactation.

We are currently developing a REST API to provide fitting services as described in this OpenAPI specification. We invite comments or suggestions.
